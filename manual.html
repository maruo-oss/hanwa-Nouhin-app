1. 初期セットアップ

1-1. スプレッドシートの作成

  1. Google Sheets で新規スプレッドシートを作成
  2. メニューの「拡張機能」→「Apps Script」をクリックしてスクリプトエディタを開く

1-2. コードの配置

スクリプトエディタに以下の3ファイルを配置します。

ファイル配置方法
Code.gsデフォルトの コード.gs を開き、内容をすべて Code.gs の内容に置き換え
index.html「+」ボタン →「HTML」を選択 → ファイル名を index（拡張子なし）にして、内容を貼り付け
dashboard.html同様に dashboard（拡張子なし）という名前でHTMLファイルを追加

1-3. appsscript.json の設定

  1. スクリプトエディタ左の歯車アイコン（プロジェクトの設定）をクリック
  2. 「「appsscript.json」マニフェストファイルをエディタで表示する」にチェック
  3. エディタに表示された appsscript.json を本プロジェクトの内容に置き換え

1-4. スクリプトプロパティの設定

「プロジェクトの設定」→「スクリプトプロパティ」に以下の2つを登録します。

プロパティ名設定する値
GEMINI_API_KEYGoogle AI Studio で取得した Gemini API キー
ROOT_FOLDER_IDPDF保存先となる Google Drive フォルダのID（URLの folders/ 以降の文字列）

1-5. デプロイ

  1. 「デプロイ」→「新しいデプロイ」をクリック
  2. 種類で「ウェブアプリ」を選択
  3. 説明文を入力（例: 納品書管理 v1）
  4. 「次のユーザーとして実行」→ 自分
  5. 「アクセスできるユーザー」→ 必要に応じて設定
  6. 「デプロイ」をクリック
  7. 表示されるURLをコピーして保存 → これがアプリのアクセスURLです
注意: コード変更後は「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョンを「新バージョン」に変更 →「デプロイ」で更新が必要です。

2. アプリへのアクセス

デプロイ時に発行されたURLにブラウザでアクセスします。

初回アクセス時にGoogleアカウントの認証許可を求められます。「許可」をクリックしてください。

認証が完了すると、以下のようなデータ管理画面が表示されます。

データ管理画面の初期状態 — 「納品書を選択してください」と表示されます

4. PDF取込（納品書の登録）

これがアプリの最も重要な機能です。納品書PDFをアップロードすると、AIが自動で内容を読み取りデータ化します。

手順

  1. ナビバー右側の「PDF取込」ボタンをクリック
  2. ファイル選択ダイアログが開くので、納品書のPDFファイルを選択（複数選択可）
  3. 自動的に以下の処理が実行されます:
    • PDFが Google Drive の指定フォルダにアップロード
    • Gemini AI がPDFの内容を解析
    • 日時・取引先・作業所・明細などのデータを自動抽出
    • スプレッドシートの「納品書データ」シートにデータを追記
  4. 処理が完了すると、左ペインのファイル一覧に新しいファイルが表示されます
ポイント: PDFファイルの命名規則に従ってファイル名をつけると、取引先名と機種が自動で抽出されます（13章参照）。
注意: GASの実行時間制限（6分）があるため、PDFは1件ずつ処理されます。大量のPDFを一度に取り込む場合は少しずつアップロードしてください。

5. データの検索・フィルタ

左ペインでファイルの絞り込みができます。

テキスト検索

  • 左上の検索ボックスにキーワードを入力
  • ファイル名・取引先名・作業所名で部分一致検索されます

ステータスフィルタ

  • 「全て」「完了」「未処理」ボタンで切り替え
  • 完了: 完了
  • 未処理: 未処理

日付範囲フィルタ

  • 「開始日」〜「終了日」を指定して、該当期間のデータに絞り込み

取引先ドロップダウン

  • 登録済みの取引先一覧から選択してフィルタリング

6. データの閲覧・編集

閲覧

  1. 左ペインのファイル一覧から、確認したいファイルをクリック
  2. 中央ペイン上部にヘッダー情報（日時、納品書No、取引先、作業所など）が表示
  3. 中央ペイン下部に明細テーブル（品目・数量・単価・金額など）が表示
  4. 右ペインに元のPDFプレビューが表示

編集

  1. 中央ペインの「編集」ボタンをクリック → 編集モードに切り替わり
  2. ヘッダー情報のフォーム、明細テーブルのセルが編集可能になります
  3. 変更したセルは黄色でハイライトされます
  4. 日時・請求計上日はカレンダーピッカーで日付を選択できます
  5. 編集が完了したら「保存」ボタンをクリック
  6. 変更を取り消す場合は「キャンセル」ボタンをクリック

ファイル単位の一括削除

  • 編集モード中に「削除」ボタンをクリックすると、そのファイルに紐づく全データを削除できます

7. 明細行の操作

明細行の編集

  • 編集モード時、各セルを直接クリックして値を変更できます
  • 区分（販売/レンタル）、機種、型式、数量、単価、金額など全項目が編集可能

明細行の削除

  • 編集モード時、各行の左端に×（削除）ボタンが表示されます
  • クリックすると、その明細行が削除されます

列幅の調整

  • 明細テーブルのヘッダー行の列境界をドラッグすると、列幅を自由に調整できます

8. PDFプレビュー

右ペインに元のPDFを表示して、データと照合確認ができます。

操作コントロール

ボタン機能

9. Excel出力

登録データをExcelファイルとしてダウンロードできます。

手順

  1. ナビバー右側の「Excel出力」ボタンをクリック
  2. スプレッドシートのデータが.xlsx形式でエクスポートされます
  3. Google Drive に保存され、ダウンロードURLが返却されます
  4. URLをクリックしてファイルをダウンロード

12. プロンプトのカスタマイズ

Gemini AIに送信するプロンプトは、スプレッドシートの「プロンプト」シートで管理されています。

変更方法

  1. スプレッドシートの「プロンプト」タブをクリック
  2. A1セルにプロンプトが格納されています
  3. 内容を直接編集して保存
メリット: コードの再デプロイなしでプロンプトの調整が可能です。AI解析の精度を向上させたい場合はここを調整してください。

13. PDFファイル名の命名規則

取引先名と機種はPDFファイル名から自動抽出されます。以下の命名規則に従うと正しく抽出されます。

フォーマット

{取引先名}_{作業所名}_{機種}_{その他}.pdf

区切り文字

以下の3種類の区切り文字に対応しています:

  • _（半角アンダースコア）
  • ＿（全角アンダースコア）
  • ‗（U+2017 ダブルアンダースコア）

抽出ルール

項目抽出位置
取引先最初の区切り文字の手前（先頭部分）
機種右から見て最初の区切りと次の区切りの間（後ろから2番目の要素）

例

ファイル名: AKｺｰﾎﾟﾚｰｼｮﾝ‗MOTHER4新築工事_JCL-015C_月例.pdf

  • 取引先 → AKｺｰﾎﾟﾚｰｼｮﾝ
  • 機種 → JCL-015C

14. トラブルシューティング

よくある問題と対処法

症状原因・対処法
アプリにアクセスできないデプロイURLが正しいか確認。コード変更後は「デプロイを管理」から新バージョンでデプロイ更新が必要
PDF取込でエラーが出るGEMINI_API_KEY と ROOT_FOLDER_ID がスクリプトプロパティに正しく設定されているか確認
PDFプレビューが表示されないGoogle Driveのアクセス権限を確認。閲覧権限が必要
AI解析の精度が低い「プロンプト」シートのA1セルでプロンプトを調整
処理がタイムアウトするGASの実行時間制限は6分。PDFを1件ずつアップロード
取引先名・機種が空になるPDFファイル名が命名規則に沿っているか確認（13章参照）
データが表示されないスプレッドシートの「納品書データ」シートにデータがあるか確認
ダッシュボードが空白データが存在するか確認。日付フィルタが適用されている場合はリセット

サポート

問題が解決しない場合は、スプレッドシートのメニュー「拡張機能」→「Apps Script」→「実行ログ」でエラーの詳細を確認してください。
