# 納品書管理 GAS ウェブアプリ

阪和建機が取引先から受け取る納品書PDFを **Gemini API（2.5 Flash）** で自動解析し、Google Sheetsにデータ蓄積・管理・可視化するGoogle Apps Script ウェブアプリケーション。

---

## ファイル構成

| ファイル | 内容 |
|---------|------|
| `appsscript.json` | GASプロジェクト設定（Drive API v3、OAuth スコープ、タイムゾーン等） |
| `Code.gs` | バックエンド全機能（ルーティング、Gemini OCR、CRUD、ダッシュボード集計、Excel出力） |
| `index.html` | ページ1: 3ペインデータ管理画面 |
| `dashboard.html` | ページ2: グラフ・集計ダッシュボード |

---

## セットアップ手順

### 1. スプレッドシート作成
- Google Sheetsで新規スプレッドシートを作成
- メニュー「拡張機能」→「Apps Script」でスクリプトエディタを開く

### 2. コード配置
- `Code.gs` の内容をスクリプトエディタに貼り付け
- `index.html` / `dashboard.html` をHTMLファイルとして追加（ファイル名は拡張子なしで `index`, `dashboard`）
- `appsscript.json` はプロジェクト設定（エディタ左の歯車アイコン → 「appsscript.json」マニフェストファイルをエディタで表示」にチェック）で編集

### 3. スクリプトプロパティ設定
「プロジェクトの設定」→「スクリプトプロパティ」に以下を設定:

| プロパティ | 内容 |
|-----------|------|
| `GEMINI_API_KEY` | Google AI Studio で取得した Gemini API キー |
| `INBOX_FOLDER_ID` | 未処理PDFを置く Google Drive フォルダID（ユーザーはここにPDFを投入） |
| `PROCESSED_FOLDER_ID` | 取込済PDFの移動先フォルダID |

> Drive 側で **未処理** と **処理済** の2つのフォルダを作成し、それぞれのID（URLの `folders/` 以降の文字列）を登録してください。

### 4. デプロイ
- 「デプロイ」→「新しいデプロイ」
- 種類: **ウェブアプリ**
- アクセスできるユーザー: 必要に応じて設定
- 「デプロイ」をクリック
- 表示されるURLからアプリにアクセス

### 5. コード変更時
コード変更後は **デプロイの更新** が必要:
1. 「デプロイ」→「デプロイを管理」
2. 鉛筆アイコン（編集）をクリック
3. バージョンを「新バージョン」に変更
4. 「デプロイ」をクリック

## 機能一覧

### データ管理画面（index.html）

3ペインレイアウト（ドラッグリサイザーで幅・高さ調整可能）:

```
┌──────────────────────────────────────────────────────────────────────┐
│ ナビバー [データ管理|ダッシュボード]  [📂PDFを入れる][🔄PDFを読み込む][📊Excel出力] │
├───────────┬──────────────────────┬─────────────────────────┤
│ 左ペイン   │ 中央ペイン（上）       │ 右ペイン                 │
│           │  ヘッダー情報フォーム   │  PDFプレビュー            │
│ ・テキスト検索│  日時/No/取引先/     │  [← ↑ ↓ → − % + ↺]    │
│ ・日付範囲  │  作業所/貸出期間/     │  ズーム・パン操作対応      │
│ ・取引先    │  請求計上日/伝票摘要   │                         │
│ ・ステータス │──────────────────────│                         │
│           │ 中央ペイン（下）       │                         │
│           │  明細テーブル          │                         │
│ [ファイル]  │  [×]区分|機種|号機|..  │                         │
├───────────┴──────────────────────┴─────────────────────────┤
│ リサイザー3本: 左右ペイン幅 + 中央上下高さ                       │
└───────────────────────────────────────────────────────────┘
```

**左ペイン（ファイルリスト）:**
- テキスト検索（ファイル名・取引先・作業所）
- 日付範囲フィルタ（開始日〜終了日）
- 取引先ドロップダウンフィルタ（データから自動生成）
- ステータスフィルタ（全て / 完了 / 未処理）

**中央ペイン:**
- 上部: ヘッダー情報フォーム（2列グリッド、日時・請求計上日はカレンダーピッカー対応）
- 下部: 明細テーブル（インライン編集対応、列幅ドラッグリサイズ対応）
- 編集 / 保存 / キャンセル切替
- 変更セルは黄色ハイライト
- 編集モード時に明細行ごとの×（削除）ボタン表示
- ファイル単位の一括削除

**右ペイン（PDFプレビュー）:**
- Google Driveプレビューをiframeで表示
- カスタムコントロール: ← → ↑ ↓ パン移動、+/- ズーム（50%〜300%）、リセット

### ダッシュボード（dashboard.html）

Google Charts による可視化（4カード構成）:

| チャート | 種類 | 内容 |
|---------|------|------|
| 月別金額推移 | ColumnChart | 月ごとの合計金額 |
| 取引先別集計 | PieChart | 取引先ごとの金額割合 |
| 機種別集計 | BarChart（横） | 機種ごとの金額 |
| 明細サマリー | HTMLテーブル | 取引先別金額一覧＋合計行 |

- 日付範囲フィルタ（開始日〜終了日）で全チャート再描画
- ウィンドウリサイズ時にチャート自動再描画

### その他機能

- **📂 PDFを入れる**: `INBOX_FOLDER_ID` に設定された Drive フォルダを新しいタブで開く
- **🔄 PDFを読み込む**: 未処理フォルダの全PDFを一括で Gemini 解析し、シートに登録＋処理済フォルダへ移動
- **重複判定**: 納品書No をキーに既存データと照合。完全一致はスキップ、差分あり（行数・金額変更など）は要確認リストとしてモーダル表示し、ユーザが「選択分を上書き」で置換
- **Excel出力**: スプレッドシートをxlsx形式でエクスポート、Driveに保存しダウンロードURL返却
- **ページ遷移**: データ管理 ↔ ダッシュボード（`target="_top"` でGASのiframe対応）

## データフロー

```
[📂 PDFを入れる] → Driveの未処理フォルダをブラウザで開く（ユーザが手動でPDFを投入）

[🔄 PDFを読み込む] → google.script.run.processInboxFolder()
  → 未処理フォルダ(INBOX_FOLDER_ID)内の全PDFを順次処理
    → 「プロンプト」シートA1からプロンプト取得
    → Gemini 2.5 Flash API でJSON抽出（temperature: 0.1）
    → ファイル名から取引先・機種を抽出
    → 納品書No で既存データと照合
      ├─ 新規: シートに追記（status = "未処理"）＋ 処理済フォルダへ移動
      ├─ 完全重複: シート不変 ＋ 処理済フォルダへ移動（スキップ報告）
      └─ 差分あり: シート不変 ＋ 処理済フォルダへ移動（要確認リストに追加）
    → 実行時間が5分を超えると安全に中断（継続は次回クリック）
  → 結果モーダル: 件数サマリ（新規/重複スキップ/要確認/エラー）+ 要確認リスト

[選択分を上書き] → google.script.run.bulkOverwriteByNo()
  → チェックされた要確認アイテムについて:
    旧シート行を削除 ＋ 旧PDFをゴミ箱へ ＋ 新データで置換
```

## PDFファイル名の命名規則

取引先名・機種はファイル名から自動抽出される:

```
{取引先名}_{作業所名}_{機種}_{その他}.pdf
```

区切り文字: `_`（半角）、`＿`（全角）、`‗`（U+2017）

- **取引先**: 最初の区切り文字の手前（先頭部分）
- **機種**: 右から見て最初の区切りと次の区切りの間（後ろから2番目の要素）

例: `AKｺｰﾎﾟﾚｰｼｮﾝ‗MOTHER4新築工事_JCL-015C_月例.pdf`
- 取引先: `AKｺｰﾎﾟﾚｰｼｮﾝ`
- 機種: `JCL-015C`

## デザイン仕様

- カラースキーム: プライマリ `#1565c0`（青）、背景 `#f4f6f8`
- フォント: Segoe UI / Hiragino Sans / Meiryo
- 全HTML/CSS/JSはインライン（GASのHtmlService制約）
- ステータスバッジ: 完了=緑、未処理=オレンジ
- 変更セルのハイライト: 黄色（`#fffde7`）

## 注意事項

- GAS実行時間制限6分: `processInboxFolder` は5分経過で安全に中断し、残りは次回の「🔄 PDFを読み込む」で継続処理
- 取込時のステータスは `未処理` として登録され、ユーザが内容確認後に手動で `完了` に変更する運用
- 重複判定キーは **納品書No**（Gemini抽出）。同じNoで中身が異なる改訂版は「選択分を上書き」で明示的に置換する
- `google.script.run` は非同期: ローディング状態管理あり
- コンテナバインドスクリプト（スプレッドシートに紐づけて使用）
- PDFプレビューは `drive.google.com/file/d/{id}/preview` を使用
- スプレッドシートから日付を取得する際は `getDisplayValues()` を使用（Dateオブジェクトのシリアライズ問題回避）
