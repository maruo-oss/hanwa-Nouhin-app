# 納品書管理 GAS ウェブアプリ

阪和建機が取引先から受け取る納品書PDFを **Gemini API（2.5 Flash）** で自動解析し、Google Sheetsにデータ蓄積・管理・可視化するGoogle Apps Script ウェブアプリケーション。

---

## ファイル構成

| ファイル | 内容 |
|---------|------|
| `appsscript.json` | GASプロジェクト設定（Drive API v3、OAuth スコープ、タイムゾーン等） |
| `Code.gs` | バックエンド全機能（ルーティング、Gemini OCR、CRUD、ダッシュボード集計、Excel出力） |
| `index.html` | ページ1: 3ペインデータ管理画面 |
| `dashboard.html` | ページ2: グラフ・集計ダッシュボード |

---

## セットアップ手順

### 1. スプレッドシート作成
- Google Sheetsで新規スプレッドシートを作成
- メニュー「拡張機能」→「Apps Script」でスクリプトエディタを開く

### 2. コード配置
- `Code.gs` の内容をスクリプトエディタに貼り付け
- `index.html` / `dashboard.html` をHTMLファイルとして追加（ファイル名は拡張子なしで `index`, `dashboard`）
- `appsscript.json` はプロジェクト設定（エディタ左の歯車アイコン → 「appsscript.json」マニフェストファイルをエディタで表示」にチェック）で編集

### 3. スクリプトプロパティ設定
「プロジェクトの設定」→「スクリプトプロパティ」に以下を設定:

| プロパティ | 内容 |
|-----------|------|
| `GEMINI_API_KEY` | Google AI Studio で取得した Gemini API キー |
| `ROOT_FOLDER_ID` | PDF保存先の Google Drive フォルダID |

### 4. デプロイ
- 「デプロイ」→「新しいデプロイ」
- 種類: **ウェブアプリ**
- アクセスできるユーザー: 必要に応じて設定
- 「デプロイ」をクリック
- 表示されるURLからアプリにアクセス

### 5. コード変更時
コード変更後は **デプロイの更新** が必要:
1. 「デプロイ」→「デプロイを管理」
2. 鉛筆アイコン（編集）をクリック
3. バージョンを「新バージョン」に変更
4. 「デプロイ」をクリック

## 機能一覧

### データ管理画面（index.html）

3ペインレイアウト（ドラッグリサイザーで幅・高さ調整可能）:

```
┌───────────────────────────────────────────────────────────┐
│ ナビバー [データ管理 | ダッシュボード]    [PDF取込] [Excel出力]  │
├───────────┬──────────────────────┬─────────────────────────┤
│ 左ペイン   │ 中央ペイン（上）       │ 右ペイン                 │
│           │  ヘッダー情報フォーム   │  PDFプレビュー            │
│ ・テキスト検索│  日時/No/取引先/     │  [← ↑ ↓ → − % + ↺]    │
│ ・日付範囲  │  作業所/貸出期間/     │  ズーム・パン操作対応      │
│ ・取引先    │  請求計上日/伝票摘要   │                         │
│ ・ステータス │──────────────────────│                         │
│           │ 中央ペイン（下）       │                         │
│           │  明細テーブル          │                         │
│ [ファイル]  │  [×]区分|機種|号機|..  │                         │
├───────────┴──────────────────────┴─────────────────────────┤
│ リサイザー3本: 左右ペイン幅 + 中央上下高さ                       │
└───────────────────────────────────────────────────────────┘
```

**左ペイン（ファイルリスト）:**
- テキスト検索（ファイル名・取引先・作業所）
- 日付範囲フィルタ（開始日〜終了日）
- 取引先ドロップダウンフィルタ（データから自動生成）
- ステータスフィルタ（全て / 完了 / 未処理）

**中央ペイン:**
- 上部: ヘッダー情報フォーム（2列グリッド）
- 下部: 明細テーブル（インライン編集対応）
- 編集 / 保存 / キャンセル切替
- 変更セルは黄色ハイライト
- 編集モード時に明細行ごとの×（削除）ボタン表示
- ファイル単位の一括削除

**右ペイン（PDFプレビュー）:**
- Google Driveプレビューをiframeで表示
- カスタムコントロール: ← → ↑ ↓ パン移動、+/- ズーム（50%〜300%）、リセット

### ダッシュボード（dashboard.html）

Google Charts による可視化（4カード構成）:

| チャート | 種類 | 内容 |
|---------|------|------|
| 月別金額推移 | ColumnChart | 月ごとの合計金額 |
| 取引先別集計 | PieChart | 取引先ごとの金額割合 |
| 機種別集計 | BarChart（横） | 機種ごとの金額 |
| 明細サマリー | HTMLテーブル | 取引先別金額一覧＋合計行 |

- 日付範囲フィルタ（開始日〜終了日）で全チャート再描画
- ウィンドウリサイズ時にチャート自動再描画

### その他機能

- **PDF取込**: ブラウザでPDF選択 → base64変換 → Drive保存 → Gemini解析 → シート書き込み
- **Excel出力**: スプレッドシートをxlsx形式でエクスポート、Driveに保存しダウンロードURL返却
- **ページ遷移**: データ管理 ↔ ダッシュボード（`target="_top"` でGASのiframe対応）

## データフロー

```
PDFアップロード(ブラウザ)
  → FileReader で base64変換
  → google.script.run.uploadAndProcessPDF()
    → Google Drive にPDF保存（ROOT_FOLDER_ID配下）
    → 「プロンプト」シートA1からプロンプト取得
    → Gemini 2.5 Flash API でJSON抽出（temperature: 0.1）
    → ファイル名から取引先を抽出
    → normalizeGeminiResponse() でスプレッドシート行に変換
    → 「納品書データ」シートに追記
  → フロントエンドにデータ返却
  → ファイルリスト更新
```

## PDFファイル名の命名規則

取引先名はファイル名の先頭部分から自動抽出される:

```
{取引先名}_{作業所名}_{機種}_{その他}.pdf
```

区切り文字: `_`（半角）、`＿`（全角）、`‗`（U+2017）

例:
- `AKｺｰﾎﾟﾚｰｼｮﾝ‗MOTHER4新築工事_JCL-015C_月例.pdf` → 取引先: `AKｺｰﾎﾟﾚｰｼｮﾝ`
- `DAIJU_プレサンス生野...pdf` → 取引先: `DAIJU`

## デザイン仕様

- カラースキーム: プライマリ `#1565c0`（青）、背景 `#f4f6f8`
- フォント: Segoe UI / Hiragino Sans / Meiryo
- 全HTML/CSS/JSはインライン（GASのHtmlService制約）
- ステータスバッジ: 完了=緑、未処理=オレンジ
- 変更セルのハイライト: 黄色（`#fffde7`）

---

## 注意事項

- GAS実行時間制限6分: PDF1件ずつ処理
- `google.script.run` は非同期: ローディング状態管理あり
- コンテナバインドスクリプト（スプレッドシートに紐づけて使用）
- PDFプレビューは `drive.google.com/file/d/{id}/preview` を使用
- スプレッドシートから日付を取得する際は `getDisplayValues()` を使用（Dateオブジェクトのシリアライズ問題回避）
